# Backend Sandbox Service Understanding

## Overview

The sandbox service (`backend-sandbox-service`) is responsible for:
- Managing sandbox definitions
- Creating and managing pools
- Allocating sandboxes to training runs
- Orchestrating sandbox lifecycle

## Key Components

### Definition Management
- **Location**: `crczp/sandbox_definition_app/lib/definitions.py`
- **Functions**:
  - `create_definition()`: Validates and creates definition
  - `get_definition()`: Retrieves topology definition from Git
  - `validate_topology_definition()`: Validates YAML structure
  - `get_containers()`: Retrieves Docker container definitions

### Pool Management
- **Location**: `crczp/sandbox_instance_app/lib/pools.py`
- **Functions**:
  - `create_pool()`: Creates pool with key pairs
  - `get_pool()`: Retrieves pool
  - Validates definition before pool creation
  - Generates SSH key pairs for management

### Sandbox Allocation
- **Location**: `crczp/sandbox_instance_app/lib/stage_handlers.py`
- **Stages**:
  1. **Allocation**: Terraform creates OpenStack resources
  2. **Networking**: ansible-stage-one configures networking
  3. **Provisioning**: Custom Ansible playbooks run
  4. **Ready**: Sandbox available for use

### Topology Processing
- **Location**: `crczp/sandbox_instance_app/lib/topology.py`
- **Classes**:
  - `Topology`: Main topology representation
  - `HostNode`: Represents a host
  - `RouterNode`: Represents a router
  - `Subnet`: Represents a network

## Sandbox Lifecycle

### 1. Definition Creation
```python
# Validates topology.yml
top_def = get_definition(definition.url, rev, config)
validate_topology_definition(top_def)
client.validate_topology_definition(top_def)
```

### 2. Pool Creation
```python
# Creates pool with management keys
pool = create_pool(data, user)
# Generates SSH keypair
private_key, public_key = generate_ssh_keypair()
# Creates OpenStack keypair
client.create_keypair(pool.ssh_keypair_name, public_key, 'ssh')
```

### 3. Sandbox Allocation
```python
# Stage 1: Allocate OpenStack stack
top_def = get_definition(definition.url, pool.rev_sha, config)
process = client.create_stack(
    top_def, 
    stack_name=stack_name,
    key_pair_name_ssh=pool.ssh_keypair_name,
    key_pair_name_cert=pool.certificate_keypair_name
)
```

### 4. Networking Stage
- Runs `ansible-stage-one` playbook
- Configures network interfaces
- Sets up routing
- Configures NAT

### 5. Provisioning Stage
- Runs custom `provisioning/playbook.yml`
- Uses inventory generated by service
- Accesses hosts via management network

## Key Data Structures

### TopologyDefinition
Parsed from `topology.yml`:
- `name`: Sandbox name
- `hosts`: List of host definitions
- `routers`: List of router definitions
- `networks`: List of network definitions
- `net_mappings`: Host-to-network mappings
- `router_mappings`: Router-to-network mappings

### Inventory
Generated for Ansible:
```yaml
all:
  hosts:
    host-name:
      ansible_host: 192.168.128.3
      ansible_user: ubuntu
      user_network_ip: 10.10.30.5
  children:
    hosts:
      hosts:
        host-name: null
    routers:
      hosts:
        router-name: null
  vars:
    global_sandbox_id: uuid
    global_sandbox_ip: 10.10.10.10
```

## Validation

### Topology Validation
- Unique names (hosts, routers, networks)
- Non-overlapping CIDRs
- Valid IP addresses (not .1 or .2)
- Valid image names
- Valid flavor names

### Container Validation
- Valid Dockerfile
- Valid docker-compose.yml
- Container definitions

## Terraform Integration

### Terraform Client
- **Location**: `crczp/sandbox_instance_app/lib/utils.py`
- **Function**: `get_terraform_client()`
- **Purpose**: Creates OpenStack resources via Terraform

### Stack Creation
1. Validates topology definition
2. Generates Terraform configuration
3. Executes Terraform apply
4. Creates VMs, networks, routers

## Ansible Integration

### Stage One (Networking)
- **Location**: `@game_builder/repos/ansible-stage-one`
- **Purpose**: Network configuration
- **Runs**: Before custom provisioning

### Custom Provisioning
- **Location**: `provisioning/playbook.yml` in definition
- **Purpose**: Host configuration
- **Runs**: After networking stage

## API Endpoints

### Definitions
- `POST /api/sandbox-definitions/`: Create definition
- `GET /api/sandbox-definitions/{id}/`: Get definition
- `POST /api/sandbox-definitions/{id}/validate/`: Validate definition

### Pools
- `POST /api/pools/`: Create pool
- `GET /api/pools/{id}/`: Get pool
- `POST /api/pools/{id}/allocate/`: Allocate sandbox

### Sandboxes
- `GET /api/sandboxes/{id}/`: Get sandbox
- `POST /api/sandboxes/{id}/lock/`: Lock sandbox
- `DELETE /api/sandboxes/{id}/`: Delete sandbox

## Error Handling

### Common Errors
- `ValidationError`: Invalid topology definition
- `GitError`: Cannot fetch from Git repository
- `StackCreationFailed`: Terraform/OpenStack error
- `InvalidTopologyDefinition`: Topology validation failed
